# 03. 日本版 Pandora 產品規格書骨架

## 1. 產品定位

**AI ソーシャルインテリジェンス平台**
= 監測 + 風險預警 + VOC 洞察 + 競品比較 + 自動報告 + 行動建議

不只是「ソーシャルリスニングツール」，而是「從發現到行動」的一條龍。理由：日本市場已存在純風險監控工具（Buzz Finder）與純深度研究工具（Quid Monitor），中間「洞察 → 行動」帶仍有空缺。

## 2. 產品分層

### 2.1 Starter – Risk Monitoring
- 對象：PR / 公關 / 風險管理 / 品牌端
- 核心任務：
  - 品牌關鍵字監測
  - 炎上 / 爆量異常告警
  - 每日摘要
  - 負面聲量趨勢
  - 夜間 / 假日異常通知
- 對標：Buzz Finder

### 2.2 Growth – VOC & Competitive Intelligence
- 對象：行銷 / 品牌 / 商品企劃
- 核心任務：
  - 品牌 / 競品聲量比較
  - 話題來源分析
  - 情緒分布
  - 痛點 / 喜好 / 使用情境萃取
  - 活動 / 新品 / 代言成效觀測
  - 月報、季報
- 對標：Pandora 主力場景

### 2.3 Enterprise – Integrated Intelligence
- 對象：大型企業 / 連鎖零售 / 跨國品牌 / 日本總部
- 核心任務：
  - 多品牌、多市場、多語言監測
  - BI 串接
  - 自有 VOC / CRM / 問卷 / Review 匯入
  - 權限控管、API
  - 24x365 代監測
  - 客製化報表與 war room
- 對標：Quid Monitor / Meltwater

## 3. P0 必做規格（MVP）

### 3.1 資料來源層
日本版 MVP 至少涵蓋：
- X
- Instagram
- Threads
- YouTube
- TikTok
- 新聞網站
- 部落格 / Web 記事
- 掲示板 / 討論區
- 評價 / Review 類站點

**在地優先順序**（依風險擴散速度）：
1. X
2. Threads
3. Instagram
4. 新聞 / 網路媒體
5. 討論區 / Review
6. YouTube / TikTok

### 3.2 關鍵字監測系統
- 主關鍵字 / 子關鍵字 / 排除字設定
- 品牌詞、商品詞、競品詞、議題詞分組
- AND / OR / NOT 布林邏輯
- 日文全半形、片假名 / 平假名 / 英文別名展開
- 品牌暱稱、自發性縮寫、自造詞收斂
- 同音異字與常見 typo 映射
- 保存查詢模板
- 一鍵複製競品 query

### 3.3 即時告警與風險分級
- 聲量異常突增告警
- 負面比例急升告警
- 高風險字詞命中告警
- 特定媒體 / KOL / 大帳號提及告警
- 夜間 / 假日告警模式
- 告警分級：Info / Warning / Critical
- 通知通道：Email / Slack / Google Chat / LINE Notify
- 告警原因摘要：為何觸發
- 告警事件頁：5 則代表貼文 + 初步風險判讀 + 建議處置

### 3.4 情緒分析與風險判讀
- 正面 / 中立 / 負面
- 負面強度分級
- 攻擊型 / 抱怨型 / 誤解型 / 爆料型 / 疑慮型分類
- 炎上前兆偵測
- 誤傳 / 謠言 / 爭議擴散類型標註
- 品牌安全風險標籤
- 可人工修正與回訓

### 3.5 去噪 / 清洗系統（MVP 核心，非附屬）
- Spam / bot / 抽獎文 / 無關重複貼文過濾
- 關鍵字同名異物排除
- 自動去除轉貼洗版
- 來源白名單 / 黑名單
- Campaign 噪音隔離
- 重複新聞稿聚合
- 類似貼文合併

> NTT 調查：使用者最不滿項目就是「不要資料的刪除 / ノイズ除去」。此模組直接影響續約率。

### 3.6 儀表板
頁面：總覽 / 風險 / VOC / 競品 / 平台別 / Campaign / Region / Top Post / KOL / 報表

每頁支援：
- 日期區間
- 平台篩選
- 品牌 / 產品 / 關鍵字篩選
- 正負情緒篩選
- 匯出圖片 / PDF / CSV

### 3.7 日報 / 週報
- 自動生成
- 可訂閱、可寄送多收件人
- 含 AI 摘要

### 3.8 日文 UI
- 全 UI、訊息、Onboarding 文案在地化
- 日本格式時間、貨幣、表單
- 無破日文（含按鈕、tooltip、錯誤訊息）

## 4. P1 第二階段規格（Growth）

### 4.1 VOC 洞察
- 自動萃取討論主題
- 自動生成 pain point / need / praise / concern
- 使用情境分類
- 人群輪廓分析
- 訴求點拆解
- 消費者常見比較基準
- 品牌好感 / 雷點摘要
- 競品優勢 / 弱點摘要
- 熱門原文引用與可追溯來源

### 4.2 競品比較
- 品牌聲量 / 情緒 / 話題占比 / 來源平台占比比較
- 時間軸比較
- 活動前後比較
- 商品線比較
- 關鍵字貢獻度比較
- Share of Voice / Share of Negative Voice

### 4.3 AI 問答分析
- 自然語句直接問：
  - 「本月負評上升原因是什麼？」
  - 「東京 20-30 代女性最常提到的 pain point 是什麼？」
  - 「競品 A 與我們相比的優勢是什麼？」
- 回答附圖表與原文依據（可追溯來源）

### 4.4 地區 / 區域分析
- 日本全國地圖熱區
- 關東 / 關西 / 九州比較
- 店舖周邊聲量
- 新店開幕觀測
- 地方限定活動反應

### 4.5 帳號 / 受眾輪廓
- 提及者類型
- 高影響力帳號
- 高轉發帳號
- 興趣標籤
- 自介詞雲
- 帳號屬性聚類

### 4.6 活動 / 新品成效追蹤
- Campaign 前中後聲量變化
- Hashtag 成效
- 正負評變化
- 話題主軸轉移
- 爆文來源與擴散路徑

### 4.7 自動報告
- 日報 / 週報 / 月報
- 異常事件報告
- 競品比較報告
- Campaign Review 報告

## 5. P2 Enterprise 規格

### 5.1 Data Uploader
- 上傳 CRM / 問卷 / 客服 / 自有評論
- 與公開輿情一起分析
- 欄位 mapping
- 評分 / 標籤 / 屬性資料支援

### 5.2 BI / API 串接
- REST API
- Webhook
- Tableau / Power BI / Domo / Looker Studio
- BigQuery / Snowflake 匯出

### 5.3 權限與組織管理
- 多品牌 / 多部門 / 多 Workspace
- Admin / Editor / Viewer
- 查詢權限限制
- 匯出權限限制
- 稽核 Log
- SSO / IP allowlist

### 5.4 Managed Service
- 24x365 人工判讀支援
- 緊急通知 SOP
- 每月顧問會議
- 分析師代做報表
- Crisis Playbook

## 6. 不放第一波的功能（明確排除）

刻意不做，避免戰線拉太開：
- 完整社群發文排程
- 完整 Influencer CRM
- 社群客服工單系統
- 超重度的媒體資料庫
- 一次上多國

理由：日本本地 Social Insight 已強在 SNS 運營一體化，Pandora 應在「風險 + 洞察 + AI 摘要 + 可行動」立場上贏，不在運營端正面對撞。

## 7. 規格優先順序總表

| 優先級 | 規格 |
| --- | --- |
| P0 | 多來源監測 / 關鍵字與排除字 / 即時告警 / 負面判讀 / 去噪 / Dashboard / 日報 / 週報 / 日文 UI |
| P1 | VOC 洞察 / 競品比較 / AI 問答 / 地區分析 / Campaign 分析 |
| P2 | Data Uploader / BI / API / SSO / 多品牌權限 / 24x365 Managed Monitoring |

## 8. 規格驗收原則

- 日本語 NLP 精度需以日本本地語料測試（含俚語、Slang、新詞、片/平/全半形混用）
- 噪音去除率需可量化（建議：spam 召回率 ≥ 90%、誤刪率 ≤ 5%）
- 告警延遲需可量化（建議：X 異常突增從事件發生到推送 ≤ 10 分鐘）
- UI 上手時間：新用戶 30 分鐘內完成首次查詢與儲存模板
- 報表自動化：月報生成不超過 1 次點擊
