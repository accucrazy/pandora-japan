03定價邏輯：關鍵字詞組 + 帳號

月費 = 基礎平台費 + 關鍵字詞組數 × 詞組單價 + 帳號數 × 帳號單價 + 選配模組
＋ 初期費用（Setup Fee，一次性）

「關鍵字詞組」= 一個品牌 / 商品 / 競品 / 議題的監測單元（含同義詞、typo、布林邏輯）。客戶感知直覺：「每多一個品牌就多一組」。

為什麼這個結構適合日本
  • 見積書文化友善：詞組 / 帳號 / 模組全明碼，利於過內部稟議。
  • 客戶語言對齊：「品牌數＝詞組數」比 Buzz Finder 的「轉送量」更可預測。
  • 自然升級路徑：帳號從 Plan 2 ¥3,500/名 降到 Plan 3 ¥2,500/名，人多自動划算。

三個核心打法
  1. 用「公式」打「黑箱」：Quid / Meltwater / Brandwatch 見積不透明，Pandora 明碼報價直接過稟議。
  2. 用「詞組」打「轉送量」：Buzz Finder 帳單難預測，「品牌數＝詞組數」是可控採購單位。
  3. 用「模組內含」打「加購」：Meltwater GenAI、Quid Generative 要加購，Pandora P2 原生內含。

05潛在成本（兩階段上市，TWD）

成本以新台幣標示，匯率 1 TWD ≒ 4.7 JPY。雲端採 GCP asia-northeast1（東京）符合 APPI。

Y1 分兩階段：先日文版、再日本渠道
  • Phase 1｜日文版 MVP（Y1 Q1–Q2）：日文 UI 本地化 + X（API + 自寫雙軌）+ 基礎日本新聞 RSS → 可賣 Plan 1。本地化由 3 FTE 平台團隊消化。
  • Phase 2｜日本在地渠道（Y1 Q3 起）：1 RD × 6 個月建 Threads / IG / 5ch / Review / 新聞擴充 → 解鎖 Plan 2。YouTube / TikTok 延至 Y2。
