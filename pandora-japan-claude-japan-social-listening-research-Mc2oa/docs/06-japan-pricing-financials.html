3.1 對標市場可以怎麼做（Pandora Playbook）

vs Zanroo（SEA 區域玩家）
  • Zanroo 在日本靠代理商、在地資料淺
  • Pandora 強打 日本在地渠道覆蓋（5ch / @cosme / 食べログ）
  • 日文 NLP 精度比 SEA 玩家高一檔
  • 話術：「日本事日本辦，不靠 SEA 模板」
vs Buzz Finder（本地入門王）
  • P1 同價 88k 円月約進場
  • 差異：AI 月報 / 情緒細分 / 主題萃取自動化
  • 通路：與公關公司合作做替換
  • 話術：「同價格、自動做完月報」
vs Social Insight（運營型）
  • 明確區隔：不做發文、只做洞察
  • P2 補 VOC / 競品深度
  • 可共存：透過 API 餵分析資料回 Social Insight
  • 話術：「運營留 SI、分析換 Pandora」
vs Quid Monitor（深分析）
  • P3 起價 60 萬円 vs Quid 100 萬円
  • UI 友善、AI 原生內含不加購
  • 攻點：中型品牌 Quid 用不完的場景
  • 話術：「深度 80%、價格 50%」
vs Meltwater / Brandwatch（國際）
  • 不跟 PR Wire 比、聚焦 social
  • 日文 NLP + 5ch / Review 差異化
  • 年約 300 萬円起 vs 對手 600 萬円+
  • 話術：「日本語精度 + 合約彈性」
三個共通打法
  • 用「公式」打「黑箱」：Pandora 明碼報價直接過稟議
  • 用「詞組」打「轉送量」：可控採購單位
  • 用「Datalake」打「Dashboard」：資料可二次分析

04Pandora 方案規格

定價公式採「基礎平台費 + 關鍵字詞組 × 單價 + 帳號 × 單價 + 選配模組」四件式結構。除此之外，Pandora 與純 Dashboard 型競品的兩個核心差異是 Token 額度制 AI 與 Datalake 原生資料資產。

4.1 產品兩大差異化：Token + Datalake

① Token 額度制 AI（每 Plan 自帶月 Token 額度）
  • 摘要、VOC 萃取、競品比較、AI 問答、月報生成 全部消耗 LLM Token
  • 每 Plan 內含 月 Token 額度，超額按量計價（明碼）
  • Plan 1：50 萬 tokens / 月；Plan 2：300 萬；Plan 3：1,500 萬
  • 超額定價：輸入 ¥0.5 / 千 tokens輸出 ¥1.5 / 千 tokens
  • 客戶好處：用多少付多少、不被「無限流量」補貼其他客戶
② Datalake 原生資料資產（不只是 Dashboard）
  • 所有 X / Threads / IG / 5ch / Review / News 資料進 GCP BigQuery Datalake（東京 Region，APPI 合規）
  • 客戶可做 ad-hoc query、歷史回溯、跨渠道關聯（非 pre-aggregated dashboard）
  • P2/P3 可透過 BI / API 直連資料湖，自家 BI / CDP / LLM pipeline 二次分析
  • 客戶的資料 永久保留（合約期內）可 re-run 不同詞組組合
  • 競品比較：Buzz Finder / Social Insight 只給 UI；Meltwater 資料需額外購買 export 權

為什麼這個結構適合日本

05研發開發成本

日本市場進入分為 3 個 Phase，總計 12 人月、一次性投入 NT$120 萬，類似製造業的「開模費」。此研發費用由騰雲（母公司）支付，Pandora Japan 不承擔此一次性成本。

5.1 開模費：NT$120 萬（三階段、12 人月，由騰雲支付）

階段內容里程碑工期人月小計
Phase 1 語言在地化 日文版 UI 全面本地化（介面 / 報表 / 通知 / 稟議範本），即可對既有資料源賣 Plan 1 日文版 MVP 上線 1 個月 1 人月 NT$10 萬

Phase 3 日本在地渠道接完 Threads / Instagram / 5ch / @cosme / 食べログ / 価格.com / 80 站新聞 RSS — 一次接完、解鎖 Plan 2 VOC + 競品洞察 全渠道覆蓋、Plan 2 可售 8–12 個月 8 人月 NT$80 萬

※ 人月單價 NT$10 萬 / 月，2 名工程師並行可壓縮至 6 個月日曆月內完成全部三階段（Phase 1 完成後即可開始 Phase 2 / 3 並行）。

「開模費」的意義（製造業類比）
  • 類似射出成型開模：NT$1.2M 是一次性 tooling 費用，由騰雲（母公司）支付，做完後日本產線可持續複用。
  • Pandora Japan 拿到的是「已開模的產品」：不用自己付研發，直接進市場銷售。
  • 騰雲的回本邏輯：NT$10 萬 / 月 × 12 個月年約 = NT$1.2M → 1 家 Plan 2/3 年約客戶就抵掉整筆開模費。
  • 後續所有新簽的日本客戶，對騰雲都是淨 ROI 貢獻。

5.3 Token 與 Datalake 的成本結構

成本項採購成本計入 Pandora 客戶方案的方式
AI / LLM Token 向 OpenAI / Anthropic 採購：輸入 $0.003 / 千、輸出 $0.015 / 千（GPT-4 級） Plan 內含月額度，超額 ¥0.5–1.5 / 千 tokens（毛利 ~60%）
BigQuery（Datalake 查詢） ~$5 / TB 掃描、儲存 $0.02 / GB-月 P1 含 6 月、P2 含 24 月、P3 無限；超額客戶延伸付儲存費
Pub/Sub（即時告警） ~$0.04 / 百萬訊息 P1 日報、P2/P3 即時告警內含
GKE（爬蟲 / API 服務） 依 CPU / Memory 使用量 共用叢集，固定成本攤提

※ Token 是變動成本（隨客戶使用量變化）、Datalake 是固定 + 少量變動成本；兩者毛利率分別 ~60% 與 ~75%，整體 COGS 毛利率維持 70%+。
